Clone the folder structure from git repository : https://github.com/sreekanthmnair/g360 
 
A) Prerequisites
1.	Connect to AWS Console programmatically “using aws configure” ,use identity which has sufficient permission
2.	Build the NGINX AMI with Packer: 
    o	To build an ami with Pre-staged nginx with static content 
    o	Run packer build nginx-ami.pkr.hcl from the prerequisite folder.
    o	Save the AMI ID generated during the build.
3.	Apply Terraform Configuration for S3 Bucket Creation:
    o	From the prerequisite folder  Run terraform init and terraform apply to create the S3 bucket and DynamoDB table for state management

B) Web Tier Stack build
4.	Set up Terraform for web stack build:
    o	Modify the terraform.tfvars file in the main folder:
    o Update ami_id with the AMI ID generated by Packer.
    o Set a unique name for s3_bucket_name to store the Terraform state file
5.	Run “Terrafom init”, make sure terraform initiated successfully without any error
6.	Run “Terrform plan” to check the environment is ready.
7.	Run “Terraform apply” to build the web tier stack.
8.	Wait for the creation, and try to access the ELB DNS Name using HTTP ( Not used https as no paid certificates provisioned)
