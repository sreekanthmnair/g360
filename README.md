Clone the folder structure from git repository : https://github.com/sreekanthmnair/g360 
 
A) Prerequisites
--------------------
1.	Connect to AWS Console programmatically “using aws configure” ,use identity which has sufficient permission
2.	Build the NGINX AMI with Packer:
3.	To build an ami with Pre-staged nginx with static content ,Run packer build nginx-ami.pkr.hcl from the prerequisite folder.
4.	Save the AMI ID generated during the build.
5.	Apply Terraform Configuration for S3 Bucket Creation,From the prerequisite folder,Run terraform init and terraform apply to create the S3 bucket and DynamoDB table for state management

B) Web Tier Stack build
-----------------------
6.	Set up Terraform for web stack build:
7.	Modify the terraform.tfvars file in the main folder:
8.	Update "ami_id" in the terraform.tfvars with the AMI ID generated by Packer.
9. Update s3_bucket_name in the terraform.tfvars to store the Terraform state file
10.	Run “Terrafom init”, make sure terraform initiated successfully without any error
11.	Run “Terrform plan” to check the environment is ready.
12.	Run “Terraform apply” to build the web tier stack.
13.	Wait for the creation, and try to access the ELB DNS Name using HTTP ( Not used https as no paid certificates provisioned)
